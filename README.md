# Delivery Defect Early-Warning System
## Last-Mile Operations Analysis · amazon_ecommerce_1M

### Project Summary
End-to-end delivery defect analysis on 10 lakh Amazon 
e-commerce transactions across 5 Indian cities. Built to 
identify leading predictors of delivery failure before 
they occur — moving from reactive reporting to proactive 
intervention.

### Key Findings
- Overall Failure Rate: 12.20% across 10 lakh orders
- Top failure reason: Time Window Missed (294,983 orders)
- Overloaded shipments fail at 20% vs 14% for normal load
- November shows highest failure rate (12.6%) — festive 
  season overload

### Tools Used
- Power BI Desktop — data cleaning, DAX measures, dashboard
- GitHub — version control and portfolio hosting

### Dataset
- amazon_ecommerce_1M.csv
- 1,000,000 rows · 20 original columns · 5 new derived columns
- Cities: Bangalore, Chennai, Delhi, Hyderabad, Mumbai

### Intervention Recommended
Based on analysis — Time Window Missed accounts for 72% 
of all failures. Narrowing delivery windows from 4 hours 
to 2 hours in overloaded zones projected to reduce failure 
rate by 25-30%, benchmarked against Amazon geocoding 
intervention (65% accuracy improvement, 2022-23).
